## Commands
- `cargo test` ŌĆö full suite (fast, <1s)
- `cargo clippy --all-targets -- -D warnings` ŌĆö required gate per task; `--all-targets` matters (also lints test code)
- `cargo run` ŌĆö agent REPL (default input runs the tool loop; /chat for plain streaming chat); `cargo run -- -p "question"` ŌĆö one-shot agent; summary to stdout, progress to stderr, exit codes 0/1/2
- `cargo run -- --auto` (or `-p ... --auto`) ŌĆö auto-approve the confirmation gate (still blocked by `auto_deny_patterns`); without it, mutating actions prompt `ņĀüņÜ®ĒĢĀĻ╣īņÜö? [y/N]` interactively, or are denied outright in `-p` non-interactive mode
- `cargo run -- eval <tasks-dir> [--repeats N] [--seed N] [--timeout-scale F]` ŌĆö eval harness (e.g. `cargo run -- eval tasks/ --repeats 3`); report table to stdout + `./.loco/eval/<stamp>/report.json` (+ per-run transcripts `run-<task>-<repeat>.jsonl`); exit 0 on normal completion regardless of pass rate, 1 on harness errors (server down, bad task defs) or a Ctrl+C partial report. Table now carries a "ņŚäĻ▓®" (strict) column and a "Ļ▒░ņ¦ō ņä▒Ļ│Ą finish" count (M6)
- `cargo run -- eval <tasks-dir> --verify` ŌĆö judge meta-test gate (no LLM/server): per task, `check` must FAIL on the pristine fixture (discriminability) and PASS after overlaying `solution/` (solvability); exit 0 only when all tasks pass both, else 1. Mutually exclusive with `--repeats`/`--seed`; `--timeout-scale` still applies. Run after ANY change to `tasks/` (M6 ┬¦4)
- `cargo run -- eval <tasks-dir> --filter <name>` (repeatable) ŌĆö exact-match task-directory filter for `eval`/`--verify`; each filter value must match Ōēź1 task or the harness errors (exit 1) ŌĆö a typo can't silently shrink a batch (M10)
- `python3 scripts/exp_metrics.py <stamp-dir>...` ŌĆö behavioral-metrics extractor for experiment eval batches (S/R errors, 2-try recovery, block/perturb-trigger counts, finish-missing streaks, stop-cause classification, merges report.json pass/strict/false-finish; ## Hard constraints (from spec)
- Edition 2024. Dependency list is fixed by the spec ŌĆö ask the user before adding any crate
- reqwest stays `default-features = false, features = ["json", "stream", "rustls-no-provider"]`; TLS crypto provider is rustls+ring (direct dep, user-approved) ŌĆö no OpenSSL and no aws-lc-sys in the graph (Windows offline builds need no cmake/NASM). `main()` installs the ring provider at startup
- Network calls only to the configured endpoint; the HTTP client uses `.no_proxy()` (corporate proxies must not capture localhost LLM traffic)
- User-facing CLI messages in Korean; identifiers and SYSTEM_PROMPT in English
- Errors: `thiserror` in `llm` module, `anyhow` at app level
- Conventional commits (subjects may be Korean)

## Architecture
- lib + thin bin: modules declared in `src/lib.rs` (`config`, `llm`, `ui`); `src/main.rs` is wiring only
- config: layered TOML ŌĆö global config_dir then `./.loco/config.toml`, later file wins, `deny_unknown_fields` rejects typos
- llm: hand-rolled SSE line parser (`llm/sse.rs`); chat retry = 3 total attempts with 200/400ms backoff, 5xx retried, 4xx immediate; `list_models` deliberately has NO retry (startup check, fail fast ŌĆö spec ┬¦9 exception)
- REPL history: index 0 is the system prompt (`/clear` truncates to 1); failed or empty streams pop the user turn so it can be retyped
- tools: `Registry::guided()` adds write_file/edit_file/run_command to the M2 three (6-tool set; `finish` stays loop-handled, not in the registry); `path::confine_for_write` is the write variant of `confine` ŌĆö target need not exist, but the deepest *existing* ancestor is canonicalized and root-checked (blocks escape via a symlinked ancestor). `edit_file`'s `search` match is a 3-stage ladder (exact ŌåÆ ignore-trailing-whitespace ŌåÆ indent-shift); 0 matches errors with the closest actual text quoted (first-line substring then char-bigram fallback, Ōēż10 lines), Ōēź2 matches errors listing up to 5 locations unless `replace_all: true` (non-overlapping greedy at stage 2, per-location indent at stage 3); success returns `render_diff_for_model`'s unified-diff body (`tools/diff.rs`) ŌĆö a one-line count header (`-R lines, +A lines[ in H hunks]`) followed by `@@`-hunk diff lines (1-line context radius), capped at `MODEL_DIFF_MAX_LINES`=15 lines total; past the cap the `@@` headers are dropped and the body ends with `[diff truncated]` (the count header still survives). Line numbers appear only in that `@@` hunk header, never as a body-line prefix ŌĆö pinned by `assert_body_lines_have_no_stray_prefix` (`src/tools/diff.rs`) on both the truncated and non-truncated paths (models copy body prefixes into their next `search`), and search==replace is an error; write_file/edit_file preserve the file's dominant EOL style (CRLF in, CRLF out, `tools/eol.rs`). Model-facing error feedback (M5): `Registry::dispatch` echoes the tool signature + received keys on serde BadArgs ("missing field"/"invalid type" only); read_file translates directory paths to a list_files hint; grep falls back to literal search on invalid regex (header carries the reason, also on 0 matches). `run_command` runs `sh -c`/`cmd /C` from project root in its own process group so timeout/cancel kills the whole tree (`kill -9 -<pgid>` / `taskkill /T /F`); UTF-8 with CP949 lossy fallback for Korean Windows consoles; output middle-truncated past 8000 bytes ŌĆö the exec mechanics live in `tools/exec.rs`, shared with eval's `check` execution. M11: Done results append one English provenance line when the command contains a real unquoted pipe (`has_unquoted_pipe` ŌĆö quote/escape-aware, `||` is not a pipe) so models don't read a pipeline's exit 0 (last command's) as test success; timeout/cancel paths get no note
- agent: confirmation gate lives inside the turn loop via the `Approver` trait (`TtyApprover`/`AutoApprover`/`NonInteractiveApprover`) ŌĆö fires only for `is_mutating()` tools with a successful `preview()`; denial feeds back to the model as a tool result. Turn parsing is salvage-tolerant (`protocol.rs` builds turns via `serde_json::Value`): fields outside `args` (action-level scalars, `args_2`, top-level non-reserved keys) are normalized into `args` (objects overwrite, scalars insert-if-absent; BTreeMap key order makes it deterministic), `ModelTurn.salvaged` triggers a one-line correction note merged into the tool result ŌĆö this drove gemma's M5 jump (285 salvages/36 runs). Repetition detection (`agent/repetition.rs`, M5): a `(tool|args, result-hash)` 8-turn window counted AFTER dispatch (denied results and summary-less `finish` included) ŌĆö 3rd identical hit injects one correction, 5th returns `AgentOutcome::RepetitionStop` (exit 2, the tool result is still pushed first); a separate same-error streak (first sentence, 3 consecutive) injects one strategy correction (edit_file/write_file ŌåÆ "rewrite with write_file", else generic). Both corrections latch once per run. M9 scaffolding: edit_file's identical-search/replace rejection now appends a prescription sentence (current code in `search`, changed code in `replace`); a dedicated 2-consecutive S/R streak correction (`SR_CORRECTION`, own latch, `agent/repetition.rs`) handles that error key exclusively ŌĆö the generic 3-streak correction is excluded for it, while the window-based `REPEAT_CORRECTION` still stacks at the 3rd identical call (intended escalation). finish discipline: a missing-summary finish 2-consecutive streak injects `FINISH_ARGS_CORRECTION` once per run (no-action turns preserve the streak; dispatched or gate-denied actions reset it); `agent/finish_nudge.rs` is an event-driven state machine ŌĆö "mutation" = successful edit_file/write_file dispatch, arms on a post-mutation `exit code: 0` run_command, counts armed-state read/grep/list_files and re-verification run_command turns in a K=4 window requiring Ōēź1 repeated call (`RepetitionTracker::seen_key`, queried before record), injects `FINISH_NUDGE` once per run; RepetitionStop takes priority on the same turn. A summary-carrying `finish` after unverified mutations is rejected once per run (`VERIFY_NUDGE`; a `run_command` Ok clears the flag regardless of exit code except an empty test run ŌĆö see M12 below). M10 adds a decoding-layer intervention alongside the M9 text corrections: an S/R streak Ōēź2 (same trigger point as `SR_CORRECTION`) overrides the request's `temperature` to 0.7 until the streak breaks (`SR_PERTURB_TEMPERATURE`, run()-scoped like `tracker`/`finish_nudge` ŌĆö no config key, no latch, re-arms if the streak re-reaches 2) ŌĆö winner of a pre-registered 3-arm experiment against a rejected alternative that blocked `edit_file` on a file after 3 cumulative S/R errors (`m10/arm-block`, preserved unmerged: it eliminated S/R-caused RepetitionStops but transferred the failure to finish discipline, a "balloon effect"); measured numbers with stamps in `docs/baselines.md` "M10 ņŗżĒŚś 1". M11 progress-state grounding: a run()-scoped `StatusNote` (`agent/status_note.rs`, pure state machine) injects a `[status]` line as the LAST note merged into tool results ŌĆö fires immediately on a successful edit_file/write_file dispatch, on a zero-mutation cadence (turns 3/5/7/10/15/20), and as unconditional pacing (15/20); renders the edited-file list (lexical dedup, cap 5 + "and N more"), verification state (last run_command exit code, overwritten to a no-exit-line notice so a stale 0 can't ground falsely), and turns used (turn = turns+1, 1-based). `session.remove_status_note()` strips the previous block before each new injection (keep-only-latest ŌĆö pack()'s elision preserves post-`</tool_result>` suffixes, so old notes are not reclaimed by shrinking; a block = the marker line + 9-space-indented continuation lines, searched only in the post-close-tag suffix so fake markers inside tool bodies are untouched, and text after the block such as a merged follow-up user request survives). Channel-less turns (length / summary-less finish / VERIFY_NUDGE rejection) carry a hit threshold over via a one-shot pending flag; RepetitionStop turns are excluded (`!stop` guard). The marker contract ŌĆö `"[status] "` prefix, 9-space continuation indent ŌĆö is shared verbatim by status_note.rs, session.rs and scripts/exp_metrics.py. Tool dispatch runs via `spawn_blocking` (synchronous tools don't block the async runtime) with a shared `cancel: AtomicBool` the REPL flips on Ctrl+C. M12 "honest harness" grounds the verification signal in parsed libtest output instead of exit code alone, and widens two M9/M10 intervention triggers ŌĆö gated by the existing `cargo test`/clippy/`eval --verify` gates plus a lightweight pre-registered regression-gate eval batch (no new effect-demonstrating experiment; results in `docs/baselines.md` "M12 ņŗżĒŚś ŌĆö ĒÜīĻĘĆ Ļ▓īņØ┤ĒŖĖ"). A pure `src/test_summary.rs::parse_test_summary` sums every `test result: ŌĆ” P passed; F failed; ŌĆ”; K filtered out ŌĆ”` section of a (possibly multi-crate/workspace) run body ŌĆö line-start-anchored so a catted log can't spoof it, `ran = passed + failed` (ignored excluded), up to 4 failed test names in encounter order ŌĆö and returns `None` when no summary line is present, the conservative fallback every consumer honors. It is parsed exactly once at the run_command dispatch site in `agent/mod.rs` and shared by three consumers: (1) `tools/run_command.rs` appends an English one-line note on an exit-0 body when `ran==0 Ōł¦ filtered_out>0` (`note: 0 tests ran (N filtered out) - cargo test filters by test NAME, not file name; this exit 0 did not verify anything` ŌĆö same append point/timeout-cancel exemption as the M11 pipe note); (2) `StatusNote`'s verification line grows from exit-code-only to 5 rules: rule 1 (`mutated_since_verify`) still wins outright; rule 2 renders up to 2 failed test names + "and N more" whenever `failed>0`, exit-code-independent (catches `cargo test 2>&1 | tail`-style masking); rule 3 renders "ran 0 tests (filter matched nothing)" on the same empty-run condition as the run_command note (different wording, non-overlapping text); rule 4 ("all P passed") additionally cross-checks `last_cmd_exit=="0"` against the same parse (blocks a truncated exit-101 body's surviving pass section from misreading as a clean pass, falling back to rule 5 otherwise); rule 5 keeps the pre-M12 `last command exited N`/`gave no exit code` text when nothing parsed; (3) `finish_nudge`'s `VerifyOk` and the `VERIFY_NUDGE` release predicate both gain `Ōł¦ ┬¼(ran==0 Ōł¦ filtered_out>0)` ŌĆö an empty run no longer arms FINISH_NUDGE nor releases VERIFY_NUDGE (both previously false-greened on a 0-test exit 0, the flagship failure motivating this milestone). `RepetitionTracker` (`agent/repetition.rs`) gains a per-file cumulative S/R counter (`sr_by_file`, keyed by `status_note::normalize`) alongside the pre-existing consecutive streak ŌĆö `SR_CORRECTION` now fires on `error_streak>=2 Ōł© cum>=2`, latched per file rather than per run (so a second file's loop still gets corrected) and capped at `MAX_SR_CORRECTIONS`=3 total per run (balloon-effect guard, per the M10 arm-block precedent); a successful edit_file/write_file dispatch (`record_mutation_ok`) clears that file's counter and latch, so a loop recurring after a real fix counts as a new incident. A parallel `badargs_streak` counts consecutive `Error: invalid arguments: missing field` bodies only (deliberately not expanded to other error kinds). `SR_PERTURB_TEMPERATURE`'s trigger widens to `sr_streak()>=2 Ōł© sr_file_streak()>=2 Ōł© badargs_streak()>=2` (mechanism/latch/lifetime unchanged ŌĆö M10's decoding-layer fix now reaching more of the loops it targeted). Dispatch (`agent/mod.rs`, before the approval gate/preview so rule 2's tool swap is judged under the resulting tool) also gains a salvage reverse rule for a model-emitted `args.tool` key: the key is always stripped before dispatch ŌĆö value equal to the action's own `tool`, or an unregistered name (`finish` included, since it's loop-handled and never registry-resident) ŌåÆ key dropped with `ARGS_TOOL_KEY_NOTE`; value naming a different *registered* tool ŌåÆ the action is redispatched as that tool with `ARGS_TOOL_SWITCH_NOTE` (the model's args apparently expressed its real intent).
