# loco — air-gapped small-model coding CLI (Rust)

Rust CLI driving a local LLM via any OpenAI-compatible API (LM Studio default `http://localhost:1234/v1`).
Spec: `docs/superpowers/specs/2026-07-02-loco-design.md`. Plans: `docs/superpowers/plans/`.
M1-M8 done. v2 baselines (M6 judge overhaul — NOT comparable to v1/M5 numbers): gemma-4-e4b 72.2% (strict 69.4%), ornith-1.0-9b 94.4% (strict 94.4%, promoted to baseline in M7's set reorg; qwen3-vl-4b retired) — details and history in `docs/baselines.md`. M7 spec: `docs/superpowers/specs/2026-07-16-m7-set-reorg-integrity-design.md`. M8 spec: `docs/superpowers/specs/2026-07-16-m8-large-repo-track-design.md` (large-repo track — see `tasks-large/` below and `docs/research/2026-07-17-m8-failure-analysis.md` for failure analysis + M9 candidates).

## Commands
- `cargo test` — full suite (fast, <1s)
- `cargo clippy --all-targets -- -D warnings` — required gate per task; `--all-targets` matters (also lints test code)
- `cargo run` — agent REPL (default input runs the tool loop; /chat for plain streaming chat); `cargo run -- -p "question"` — one-shot agent; summary to stdout, progress to stderr, exit codes 0/1/2
- `cargo run -- --auto` (or `-p ... --auto`) — auto-approve the confirmation gate (still blocked by `auto_deny_patterns`); without it, mutating actions prompt `적용할까요? [y/N]` interactively, or are denied outright in `-p` non-interactive mode
- `cargo run -- eval <tasks-dir> [--repeats N] [--seed N] [--timeout-scale F]` — eval harness (e.g. `cargo run -- eval tasks/ --repeats 3`); report table to stdout + `./.loco/eval/<stamp>/report.json` (+ per-run transcripts `run-<task>-<repeat>.jsonl`); exit 0 on normal completion regardless of pass rate, 1 on harness errors (server down, bad task defs) or a Ctrl+C partial report. Table now carries a "엄격" (strict) column and a "거짓 성공 finish" count (M6)
- `cargo run -- eval <tasks-dir> --verify` — judge meta-test gate (no LLM/server): per task, `check` must FAIL on the pristine fixture (discriminability) and PASS after overlaying `solution/` (solvability); exit 0 only when all tasks pass both, else 1. Mutually exclusive with `--repeats`/`--seed`; `--timeout-scale` still applies. Run after ANY change to `tasks/` (M6 §4)
- Session transcripts: one JSONL file per run at `./.loco/sessions/*.jsonl` (best-effort; `.loco/.gitignore` keeps them out of commits)
- Tunable config (defaults): `max_turns`=25, `max_output_tokens`=2048, `context_tokens`=8192, `command_timeout_secs`=60, `auto_deny_patterns`; unknown keys rejected (`deny_unknown_fields`)
- Server-down smoke: temp dir with `.loco/config.toml` pointing `base_url` at `http://127.0.0.1:1/v1` (port 1 is reliably closed; ephemeral ports may be occupied)

## Hard constraints (from spec)
- Edition 2024. Dependency list is fixed by the spec — ask the user before adding any crate
- reqwest stays `default-features = false, features = ["json", "stream", "rustls-no-provider"]`; TLS crypto provider is rustls+ring (direct dep, user-approved) — no OpenSSL and no aws-lc-sys in the graph (Windows offline builds need no cmake/NASM). `main()` installs the ring provider at startup
- Network calls only to the configured endpoint; the HTTP client uses `.no_proxy()` (corporate proxies must not capture localhost LLM traffic)
- User-facing CLI messages in Korean; identifiers and SYSTEM_PROMPT in English
- Errors: `thiserror` in `llm` module, `anyhow` at app level
- Conventional commits (subjects may be Korean)

## Architecture
- lib + thin bin: modules declared in `src/lib.rs` (`config`, `llm`, `ui`); `src/main.rs` is wiring only
- config: layered TOML — global config_dir then `./.loco/config.toml`, later file wins, `deny_unknown_fields` rejects typos
- llm: hand-rolled SSE line parser (`llm/sse.rs`); chat retry = 3 total attempts with 200/400ms backoff, 5xx retried, 4xx immediate; `list_models` deliberately has NO retry (startup check, fail fast — spec §9 exception)
- REPL history: index 0 is the system prompt (`/clear` truncates to 1); failed or empty streams pop the user turn so it can be retyped
- tools: `Registry::guided()` adds write_file/edit_file/run_command to the M2 three (6-tool set; `finish` stays loop-handled, not in the registry); `path::confine_for_write` is the write variant of `confine` — target need not exist, but the deepest *existing* ancestor is canonicalized and root-checked (blocks escape via a symlinked ancestor). `edit_file`'s `search` match is a 3-stage ladder (exact → ignore-trailing-whitespace → indent-shift); 0 matches errors with the closest actual text quoted (first-line substring then char-bigram fallback, ≤10 lines), ≥2 matches errors listing up to 5 locations unless `replace_all: true` (non-overlapping greedy at stage 2, per-location indent at stage 3); success returns ±3 lines of post-edit context (line numbers in the header only — models copy body prefixes into their next `search`), and search==replace is an error; write_file/edit_file preserve the file's dominant EOL style (CRLF in, CRLF out, `tools/eol.rs`). Model-facing error feedback (M5): `Registry::dispatch` echoes the tool signature + received keys on serde BadArgs ("missing field"/"invalid type" only); read_file translates directory paths to a list_files hint; grep falls back to literal search on invalid regex (header carries the reason, also on 0 matches). `run_command` runs `sh -c`/`cmd /C` from project root in its own process group so timeout/cancel kills the whole tree (`kill -9 -<pgid>` / `taskkill /T /F`); UTF-8 with CP949 lossy fallback for Korean Windows consoles; output middle-truncated past 8000 bytes — the exec mechanics live in `tools/exec.rs`, shared with eval's `check` execution
- agent: confirmation gate lives inside the turn loop via the `Approver` trait (`TtyApprover`/`AutoApprover`/`NonInteractiveApprover`) — fires only for `is_mutating()` tools with a successful `preview()`; denial feeds back to the model as a tool result. Turn parsing is salvage-tolerant (`protocol.rs` builds turns via `serde_json::Value`): fields outside `args` (action-level scalars, `args_2`, top-level non-reserved keys) are normalized into `args` (objects overwrite, scalars insert-if-absent; BTreeMap key order makes it deterministic), `ModelTurn.salvaged` triggers a one-line correction note merged into the tool result — this drove gemma's M5 jump (285 salvages/36 runs). Repetition detection (`agent/repetition.rs`, M5): a `(tool|args, result-hash)` 8-turn window counted AFTER dispatch (denied results and summary-less `finish` included) — 3rd identical hit injects one correction, 5th returns `AgentOutcome::RepetitionStop` (exit 2, the tool result is still pushed first); a separate same-error streak (first sentence, 3 consecutive) injects one strategy correction (edit_file/write_file → "rewrite with write_file", else generic). Both corrections latch once per run. A summary-carrying `finish` after unverified mutations is rejected once per run (`VERIFY_NUDGE`; any `run_command` Ok clears the flag regardless of exit code). Tool dispatch runs via `spawn_blocking` (synchronous tools don't block the async runtime) with a shared `cancel: AtomicBool` the REPL flips on Ctrl+C
- session: `Session` (src/session.rs) owns chat history + the JSONL transcript together — every push records both. `pack()` is the spec §6 budget: elide oldest tool-result bodies first, then drop oldest user+assistant pairs atomically (system prompt and final message preserved), mutating stored history in place (transcript keeps unabridged originals). Context-overflow 400 (heuristic: body contains "context") packs at a shrinking budget and retries, up to 2 shrinks per `run()` call before propagating — takes priority over the blind 400 fallback ladder, which doesn't fire on overflow
- `--auto`: `AutoApprover` approves everything except `run_command` matching `auto_deny_patterns` (cross-platform default list — `rm -rf`, `sudo`, `git push`, etc., configurable in TOML), blocked only in auto mode; interactive mode shows a warning line instead
- `eval` (`src/eval/`): in-process harness — per task×repeat: fixture → temp sandbox → `Agent::run` with `AutoApprover` + seed(base_seed+repeat) → protected paths re-synced from the fixture (deletes agent-added files — anti reward-hack; `.cargo` is implicitly protected; the tripwire's effective check is `temp_dir/.cargo` only (the sandbox's parent IS temp_dir — detection only, no cleanup: remove a leftover `${TMPDIR}/.cargo` manually). M7 adds change-detection snapshots (`eval/integrity.rs`): `$CARGO_HOME` config.toml/config plus `.cargo/config*` in every ancestor above canonicalized temp_dir are recorded at harness start and re-compared before each check — any state transition aborts the harness. Still out of scope (backlog): pre-existing poisoned configs (CARGO_HOME isolation) and cargo-binary replacement (`$CARGO_HOME/bin`, `~/.rustup` — PATH pinning/hashing)) → the `check` command's exit code decides pass; `check` always runs, regardless of outcome. Timeout math is clamped (`scaled_timeout`: saturating, cap 3600s — huge `timeout_secs`/non-finite scale can't panic `from_secs_f64`); report.json snapshots the effective config (`effective_config`: base_url, temperature, context/output tokens, max_turns, command timeout, loco version) for cross-machine comparison. An LLM error aborts the whole harness; Ctrl+C writes a partial report (`interrupted: true`, exit 1). Timeouts: `agent::bounded::run_bounded` sets `ToolCtx.cancel` then waits `cancel_grace` so `run_command` can kill its process group before giving up (no orphans) — the same helper wires `-p`'s Ctrl+C handling (exit 2). `AgentOutcome::Cancelled` is a loop-top cancel check, keeping the grace window fast. `report.json` also aggregates `passed_count`/`passed_strict_count` (passed ∧ `outcome==finished`)/`false_finish_count` (`outcome==finished` ∧ !passed) per task and at the top level — all `_count`-suffixed additions, existing keys (`pass_rate`, `total_pass_rate`, per-run `passed`) unchanged (M6 §5), plus top-level `avg_duration_secs` (run-weighted mean of per-run agent durations — excludes check/sandbox overhead, so it differs from wall-clock `duration_secs`/runs; M7 §4). `verify` (`src/eval/verify.rs`, M6 §4) is a separate LLM-free gate: each task dir is `task.toml` + `fixture/` + `solution/`, where `solution/` is a file-level overlay of only the changed/added files (no delete/rename expression); `overlay_tree` and the eval `sync_protected` restore both do read+write (NOT `fs::copy`, whose mtime preservation on macOS causes stale test-binary judging); verify rejects a `solution/` that overlaps any protected path (incl. implicit `.cargo`), and `cargo_tripwire` runs in verify too
- `tasks/`: 12 zero-dependency cargo-crate fixtures (one dir each — `task.toml` + `fixture/` + `solution/`); all currently use `check = "cargo test"`, `protected = ["tests", "Cargo.toml"]`; `tasks/.gitattributes` pins the CRLF fixture+solution byte-exact; every fixture MUST carry a `/target` `.gitignore` — the walker honors .gitignore without git (`require_git(false)`), and without it a post-`cargo test` bare `list_files` returns ~14KB of target/ paths as an unshrinkable final message → context-overflow 400. Every task carries a `solution/` reference overlay (the `--verify` gate's solvability half); answer-format judges (find-definition, count-usages) normalize before comparing — trim → strip matched surrounding quotes → path/int canonicalize — and self-test that ladder with fixed accept/reject cases living in the same `tests/*.rs` file (so `check` exercises the ladder on both eval and verify runs, M6 §3)
- `tasks-large/` (M8): 3 eval tasks on a shared 5-crate `inv-*` workspace fixture (10-20K LOC) — `find-definition-large` measures locating a definition buried past the system-prompt tree's depth cap; `fix-monthly-total` measures picking the right one of two duplicate summation functions (v1/v2) to patch amid false comments/FIXMEs; `update-vat-rate` measures propagating a rate constant across 4 scattered call sites without missing one. Run `cargo run -- eval tasks-large --verify` after ANY tasks-large change (3/3 expected — same discipline as `tasks/`). `tasks-large/README.md` sits outside `fixture/` (never copied into the sandbox, so it's model-invisible) and holds the 11-trap ledger plus the drift procedure: the base is `find-definition-large/fixture`, with `fix-monthly-total`/`update-vat-rate` maintained as `rsync` copies of it (task-specific bug/change layered on top) — any base edit must be re-synced to both per that procedure. M8 measured results (3 tasks × 3 repeats, seed 0): gemma-4-e4b 44.4% / ornith-1.0-9b 55.6% at 8K context, ornith-1.0-9b 88.9% at 32K context (strict unchanged at 44.4% — context rescues lenient pass but not finish discipline) — both far below the small-repo `tasks/` baselines (gemma 72.2%/ornith 94.4%), confirming the large-repo track measures a distinct failure axis; per-run classification, trap-hit attribution, and M9 requirement candidates in `docs/research/2026-07-17-m8-failure-analysis.md`

## Notes
- `.superpowers/` is git-ignored scratch (subagent-driven-development progress ledger lives there); `git clean -fdx` destroys it
- Remote: `origin` = github.com/SeonggukJeong/loco.git — push only when the user asks
- Live smoke/eval needs LM Studio with a model loaded at context length ≥ `context_tokens` (8192) — smaller loaded ctx makes LM Studio 400 ("n_keep > context length") mid-eval, aborting the harness. `model = ""` auto-selects the FIRST `/v1/models` entry; LM Studio sorts loaded models first, so unload the old model when switching for comparison runs. `curl localhost:1234/api/v0/models` shows loaded state + context length
- Eval measurement protocol: a full `eval tasks --repeats 3` run takes ~45-75 min per model; do NOT run cargo build/test concurrently with a live measurement — CPU contention skews the timing-sensitive judgments. report.json nests runs per task (`tasks[].runs[]` with passed/outcome per run), not a flat run list; per-batch metric grep recipes are in docs/baselines.md "M5 경과"
- `TtyApprover` is intentionally synchronous-blocking on `readline()`, not polled/async — an async stdin reader left running during the prompt would steal the Ctrl+C keystroke meant for the REPL's `select!`, orphaning the reader task
- Small local models (e.g. gemma-4-e4b) can loop on `finish_reason: length` for long outputs (summaries, etc.) until `max_turns` → exit 2 (spec §3 v1 blind spot, bounded, not a hang). Mitigate: raise `max_output_tokens` in `./.loco/config.toml` (git-ignored via `.loco/.gitignore` = `*`, so local-only) or load a larger model
- eval tests are `#[cfg(unix)]`-gated where they need a shell or symlinks: all of `src/eval/mod.rs`'s integration tests (checks run via `sh -c`) and 2 of 8 `src/eval/sandbox.rs` tests (unix symlink APIs); the `tasks/` fixture crates are not cargo workspace members, so root `cargo build`/`test`/`clippy` ignore them
