# loco ‚Äî air-gapped small-model coding CLI (Rust)

Rust CLI driving a local LLM via any OpenAI-compatible API (LM Studio default `http://localhost:1234/v1`).
Spec: `docs/superpowers/specs/2026-07-02-loco-design.md`. Plans: `docs/superpowers/plans/`.
M1-M5 done (guided agent + eval harness + baselines + scaffolding improvements). Pass rates (`eval tasks --repeats 3`, seed 0, ctx 8192): gemma-4-e4b 11.1% ‚Üí 66.7%, qwen3-vl-4b 33.3% ‚Üí 50.0% after M5 ‚Äî per-batch history and final analysis in `docs/baselines.md`. M5 spec: `docs/superpowers/specs/2026-07-12-m5-scaffolding-design.md`.

## Commands
- `cargo test` ‚Äî full suite (fast, <1s)
- `cargo clippy --all-targets -- -D warnings` ‚Äî required gate per task; `--all-targets` matters (also lints test code)
- `cargo run` ‚Äî agent REPL (default input runs the tool loop; /chat for plain streaming chat); `cargo run -- -p "question"` ‚Äî one-shot agent; summary to stdout, progress to stderr, exit codes 0/1/2
- `cargo run -- --auto` (or `-p ... --auto`) ‚Äî auto-approve the confirmation gate (still blocked by `auto_deny_patterns`); without it, mutating actions prompt `ÏÝÅÏö©ÌïÝÍπåÏöî? [y/N]` interactively, or are denied outright in `-p` non-interactive mode
- `cargo run -- eval <tasks-dir> [--repeats N] [--seed N] [--timeout-scale F]` ‚Äî eval harness (e.g. `cargo run -- eval tasks/ --repeats 3`); report table to stdout + `./.loco/eval/<stamp>/report.json` (+ per-run transcripts `run-<task>-<repeat>.jsonl`); exit 0 on normal completion regardless of pass rate, 1 on harness errors (server down, bad task defs) or a Ctrl+C partial report. Table now carries a "ÏóÑÍ≤©" (strict) column and a "Í±∞Ïßì ÏÑ±Í≥µ finish" count (M6)
- `cargo run -- eval <tasks-dir> --verify` ‚Äî judge meta-test gate (no LLM/server): per task, `check` must FAIL on the pristine fixture (discriminability) and PASS after overlaying `solution/` (solvability); exit 0 only when all tasks pass both, else 1. Mutually exclusive with `--repeats`/`--seed`; `--timeout-scale` still applies. Run after ANY change to `tasks/` (M6 ¬ß4)
- Session transcripts: one JSONL file per run at `./.loco/sessions/*.jsonl` (best-effort; `.loco/.gitignore` keeps them out of commits)
- Tunable config (defaults): `max_turns`=25, `max_output_tokens`=2048, `context_tokens`=8192, `command_timeout_secs`=60, `auto_deny_patterns`; unknown keys rejected (`deny_unknown_fields`)
- Server-down smoke: temp dir with `.loco/config.toml` pointing `base_url` at `http://127.0.0.1:1/v1` (port 1 is reliably closed; ephemeral ports may be occupied)

## Hard constraints (from spec)
- Edition 2024. Dependency list is fixed by the spec ‚Äî ask the user before adding any crate
- reqwest stays `default-features = false, features = ["json", "stream", "rustls-no-provider"]`; TLS crypto provider is rustls+ring (direct dep, user-approved) ‚Äî no OpenSSL and no aws-lc-sys in the graph (Windows offline builds need no cmake/NASM). `main()` installs the ring provider at startup
- Network calls only to the configured endpoint; the HTTP client uses `.no_proxy()` (corporate proxies must not capture localhost LLM traffic)
- User-facing CLI messages in Korean; identifiers and SYSTEM_PROMPT in English
- Errors: `thiserror` in `llm` module, `anyhow` at app level
- Conventional commits (subjects may be Korean)

## Architecture
- lib + thin bin: modules declared in `src/lib.rs` (`config`, `llm`, `ui`); `src/main.rs` is wiring only
- config: layered TOML ‚Äî global config_dir then `./.loco/config.toml`, later file wins, `deny_unknown_fields` rejects typos
- llm: hand-rolled SSE line parser (`llm/sse.rs`); chat retry = 3 total attempts with 200/400ms backoff, 5xx retried, 4xx immediate; `list_models` deliberately has NO retry (startup check, fail fast ‚Äî spec ¬ß9 exception)
- REPL history: index 0 is the system prompt (`/clear` truncates to 1); failed or empty streams pop the user turn so it can be retyped
- tools: `Registry::guided()` adds write_file/edit_file/run_command to the M2 three (6-tool set; `finish` stays loop-handled, not in the registry); `path::confine_for_write` is the write variant of `confine` ‚Äî target need not exist, but the deepest *existing* ancestor is canonicalized and root-checked (blocks escape via a symlinked ancestor). `edit_file`'s `search` match is a 3-stage ladder (exact ‚Üí ignore-trailing-whitespace ‚Üí indent-shift); 0 matches errors with the closest actual text quoted (first-line substring then char-bigram fallback, ‚â§10 lines), ‚â•2 matches errors listing up to 5 locations unless `replace_all: true` (non-overlapping greedy at stage 2, per-location indent at stage 3); success returns ¬±3 lines of post-edit context (line numbers in the header only ‚Äî models copy body prefixes into their next `search`), and search==replace is an error; write_file/edit_file preserve the file's dominant EOL style (CRLF in, CRLF out, `tools/eol.rs`). Model-facing error feedback (M5): `Registry::dispatch` echoes the tool signature + received keys on serde BadArgs ("missing field"/"invalid type" only); read_file translates directory paths to a list_files hint; grep falls back to literal search on invalid regex (header carries the reason, also on 0 matches). `run_command` runs `sh -c`/`cmd /C` from project root in its own process group so timeout/cancel kills the whole tree (`kill -9 -<pgid>` / `taskkill /T /F`); UTF-8 with CP949 lossy fallback for Korean Windows consoles; output middle-truncated past 8000 bytes ‚Äî the exec mechanics live in `tools/exec.rs`, shared with eval's `check` execution
- agent: confirmation gate lives inside the turn loop via the `Approver` trait (`TtyApprover`/`AutoApprover`/`NonInteractiveApprover`) ‚Äî fires only for `is_mutating()` tools with a successful `preview()`; denial feeds back to the model as a tool result. Turn parsing is salvage-tolerant (`protocol.rs` builds turns via `serde_json::Value`): fields outside `args` (action-level scalars, `args_2`, top-level non-reserved keys) are normalized into `args` (objects overwrite, scalars insert-if-absent; BTreeMap key order makes it deterministic), `ModelTurn.salvaged` triggers a one-line correction note merged into the tool result ‚Äî this drove gemma's M5 jump (285 salvages/36 runs). Repetition detection (`agent/repetition.rs`, M5): a `(tool|args, result-hash)` 8-turn window counted AFTER dispatch (denied results and summary-less `finish` included) ‚Äî 3rd identical hit injects one correction, 5th returns `AgentOutcome::RepetitionStop` (exit 2, the tool result is still pushed first); a separate same-error streak (first sentence, 3 consecutive) injects one strategy correction (edit_file/write_file ‚Üí "rewrite with write_file", else generic). Both corrections latch once per run. A summary-carrying `finish` after unverified mutations is rejected once per run (`VERIFY_NUDGE`; any `run_command` Ok clears the flag regardless of exit code). Tool dispatch runs via `spawn_blocking` (synchronous tools don't block the async runtime) with a shared `cancel: AtomicBool` the REPL flips on Ctrl+C
- session: `Session` (src/session.rs) owns chat history + the JSONL transcript together ‚Äî every push records both. `pack()` is the spec ¬ß6 budget: elide oldest tool-result bodies first, then drop oldest user+assistant pairs atomically (system prompt and final message preserved), mutating stored history in place (transcript keeps unabridged originals). Context-overflow 400 (heuristic: body contains "context") packs at a shrinking budget and retries, up to 2 shrinks per `run()` call before propagating ‚Äî takes priority over the blind 400 fallback ladder, which doesn't fire on overflow
- `--auto`: `AutoApprover` approves everything except `run_command` matching `auto_deny_patterns` (cross-platform default list ‚Äî `rm -rf`, `sudo`, `git push`, etc., configurable in TOML), blocked only in auto mode; interactive mode shows a warning line instead
- `eval` (`src/eval/`): in-process harness ‚Äî per task√órepeat: fixture ‚Üí temp sandbox ‚Üí `Agent::run` with `AutoApprover` + seed(base_seed+repeat) ‚Üí protected paths re-synced from the fixture (deletes agent-added files ‚Äî anti reward-hack; `.cargo` is implicitly protected, and a `.cargo` anywhere above the sandbox aborts the harness ‚Äî tripwire detects but does not clean up, so a leftover `${TMPDIR}/.cargo` must be removed manually; `$CARGO_HOME`/home-dir vectors remain unblocked) ‚Üí the `check` command's exit code decides pass; `check` always runs, regardless of outcome. Timeout math is clamped (`scaled_timeout`: saturating, cap 3600s ‚Äî huge `timeout_secs`/non-finite scale can't panic `from_secs_f64`); report.json snapshots the effective config (`effective_config`: base_url, temperature, context/output tokens, max_turns, command timeout, loco version) for cross-machine comparison. An LLM error aborts the whole harness; Ctrl+C writes a partial report (`interrupted: true`, exit 1). Timeouts: `agent::bounded::run_bounded` sets `ToolCtx.cancel` then waits `cancel_grace` so `run_command` can kill its process group before giving up (no orphans) ‚Äî the same helper wires `-p`'s Ctrl+C handling (exit 2). `AgentOutcome::Cancelled` is a loop-top cancel check, keeping the grace window fast. `report.json` also aggregates `passed_count`/`passed_strict_count` (passed ‚àß `outcome==finished`)/`false_finish_count` (`outcome==finished` ‚àß !passed) per task and at the top level ‚Äî all `_count`-suffixed additions, existing keys (`pass_rate`, `total_pass_rate`, per-run `passed`) unchanged (M6 ¬ß5). `verify` (`src/eval/verify.rs`, M6 ¬ß4) is a separate LLM-free gate: each task dir is `task.toml` + `fixture/` + `solution/`, where `solution/` is a file-level overlay of only the changed/added files (no delete/rename expression); `overlay_tree` and the eval `sync_protected` restore both do read+write (NOT `fs::copy`, whose mtime preservation on macOS causes stale test-binary judging); verify rejects a `solution/` that overlaps any protected path (incl. implicit `.cargo`), and `cargo_tripwire` runs in verify too
- `tasks/`: 12 zero-dependency cargo-crate fixtures (one dir each ‚Äî `task.toml` + `fixture/` + `solution/`); all currently use `check = "cargo test"`, `protected = ["tests", "Cargo.toml"]`; `tasks/.gitattributes` pins the CRLF fixture+solution byte-exact; every fixture MUST carry a `/target` `.gitignore` ‚Äî the walker honors .gitignore without git (`require_git(false)`), and without it a post-`cargo test` bare `list_files` returns ~14KB of target/ paths as an unshrinkable final message ‚Üí context-overflow 400. Every task carries a `solution/` reference overlay (the `--verify` gate's solvability half); answer-format judges (find-definition, count-usages) normalize before comparing ‚Äî trim ‚Üí strip matched surrounding quotes ‚Üí path/int canonicalize ‚Äî and self-test that ladder with fixed accept/reject cases living in the same `tests/*.rs` file (so `check` exercises the ladder on both eval and verify runs, M6 ¬ß3)

## Notes
- `.superpowers/` is git-ignored scratch (subagent-driven-development progress ledger lives there); `git clean -fdx` destroys it
- No git remote ‚Äî local-only repo
- Live smoke/eval needs LM Studio with a model loaded at context length ‚â• `context_tokens` (8192) ‚Äî smaller loaded ctx makes LM Studio 400 ("n_keep > context length") mid-eval, aborting the harness. `model = ""` auto-selects the FIRST `/v1/models` entry; LM Studio sorts loaded models first, so unload the old model when switching for comparison runs. `curl localhost:1234/api/v0/models` shows loaded state + context length
- Eval measurement protocol: a full `eval tasks --repeats 3` run takes ~45-75 min per model; do NOT run cargo build/test concurrently with a live measurement ‚Äî CPU contention skews the timing-sensitive judgments. report.json nests runs per task (`tasks[].runs[]` with passed/outcome per run), not a flat run list; per-batch metric grep recipes are in docs/baselines.md "M5 Í≤ΩÍ≥º"
- `TtyApprover` is intentionally synchronous-blocking on `readline()`, not polled/async ‚Äî an async stdin reader left running during the prompt would steal the Ctrl+C keystroke meant for the REPL's `select!`, orphaning the reader task
- Small local models (e.g. gemma-4-e4b) can loop on `finish_reason: length` for long outputs (summaries, etc.) until `max_turns` ‚Üí exit 2 (spec ¬ß3 v1 blind spot, bounded, not a hang). Mitigate: raise `max_output_tokens` in `./.loco/config.toml` (git-ignored via `.loco/.gitignore` = `*`, so local-only) or load a larger model
- eval tests are `#[cfg(unix)]`-gated where they need a shell or symlinks: all of `src/eval/mod.rs`'s integration tests (checks run via `sh -c`) and 2 of 8 `src/eval/sandbox.rs` tests (unix symlink APIs); the `tasks/` fixture crates are not cargo workspace members, so root `cargo build`/`test`/`clippy` ignore them
