## Commands
- `cargo test` ‚Äî full suite (fast, <1s)
- `cargo clippy --all-targets -- -D warnings` ‚Äî required gate per task; `--all-targets` matters (also lints test code)
- `cargo run` ‚Äî agent REPL (default input runs the tool loop; /chat for plain streaming chat); `cargo run -- -p "question"` ‚Äî one-shot agent; summary to stdout, progress to stderr, exit codes 0/1/2
- `cargo run -- --auto` (or `-p ... --auto`) ‚Äî auto-approve the confirmation gate (still blocked by `auto_deny_patterns`); without it, mutating actions prompt `ÏÝÅÏö©ÌïÝÍπåÏöî? [y/N]` interactively, or are denied outright in `-p` non-interactive mode
- `cargo run -- eval <tasks-dir> [--repeats N] [--seed N] [--timeout-scale F]` ‚Äî eval harness (e.g. `cargo run -- eval tasks/ --repeats 3`); report table to stdout + `./.loco/eval/<stamp>/report.json` (+ per-run transcripts `run-<task>-<repeat>.jsonl`); exit 0 on normal completion regardless of pass rate, 1 on harness errors (server down, bad task defs) or a Ctrl+C partial report. Table now carries a "ÏóÑÍ≤©" (strict) column and a "Í±∞Ïßì ÏÑ±Í≥µ finish" count (M6)
- `cargo run -- eval <tasks-dir> --verify` ‚Äî judge meta-test gate (no LLM/server): per task, `check` must FAIL on the pristine fixture (discriminability) and PASS after overlaying `solution/` (solvability); exit 0 only when all tasks pass both, else 1. Mutually exclusive with `--repeats`/`--seed`; `--timeout-scale` still applies. Run after ANY change to `tasks/` (M6 ¬ß4)
- `cargo run -- eval <tasks-dir> --filter <name>` (repeatable) ‚Äî exact-match task-directory filter for `eval`/`--verify`; each filter value must match ‚â•1 task or the harness errors (exit 1) ‚Äî a typo can't silently shrink a batch (M10)

## Hard constraints (from spec)
- Edition 2024. Dependency list is fixed by the spec ‚Äî ask the user before adding any crate
- reqwest stays `default-features = false, features = ["json", "stream", "rustls-no-provider"]`; TLS crypto provider is rustls+ring (direct dep, user-approved) ‚Äî no OpenSSL and no aws-lc-sys in the graph (Windows offline builds need no cmake/NASM). `main()` installs the ring provider at startup

## Architecture
- lib + thin bin: modules declared in `src/lib.rs` (`config`, `llm`, `ui`); `src/main.rs` is wiring only
- config: layered TOML ‚Äî global config_dir then `./.loco/config.toml`, later file wins, `deny_unknown_fields` rejects typos
- llm: hand-rolled SSE line parser (`llm/sse.rs`); chat retry = 3 total attempts with 200/400ms backoff, 5xx retried, 4xx immediate; `list_models` deliberately has NO retry (startup check, fail fast ‚Äî spec ¬ß9 exception)
- REPL history: index 0 is the system prompt (`/clear` truncates to 1); failed or empty streams pop the user turn so it can be retyped
- tools: `Registry::guided(repo_notes: bool)` adds write_file/edit_file/run_command to the M2 three (6-tool set when `repo_notes=false`; with `repo_notes=true` also registers `update_repo_notes` for hierarchical `.loco/notes/` ‚Äî M16; `finish` stays loop-handled, not in the registry); `path::confine_for_write` is the write variant of `confine` ‚Äî target need not exist, but the deepest *existing* ancestor is canonicalized and root-checked (blocks escape via a symlinked ancestor). `edit_file`'s `search` match is a 3-stage ladder (exact ‚Üí ignore-trailing-whitespace ‚Üí indent-shift); 0 matches errors with the closest actual text quoted (first-line substring then char-bigram fallback, ‚â§10 lines), ‚â•2 matches errors listing up to 5 locations unless `replace_all: true` (non-overlapping greedy at stage 2, per-location indent at stage 3); success returns `render_diff_for_model`'s unified-diff body (`tools/diff.rs`) ‚Äî a one-line count header (`-R lines, +A lines[ in H hunks]`) followed by `@@`-hunk diff lines (1-line context radius), capped at `MODEL_DIFF_MAX_LINES`=15 lines total; past the cap the `@@` headers are dropped and the body ends with `[diff truncated]` (the count header still survives). Line numbers appear only in that `@@` hunk header, never as a body-line prefix ‚Äî pinned by `assert_body_lines_have_no_stray_prefix` (`src/tools/diff.rs`) on both the truncated and non-truncated paths (models copy body prefixes into their next `search`), and search==replace is an error; write_file/edit_file preserve the file's dominant EOL style (CRLF in, CRLF out, `tools/eol.rs`). Model-facing error feedback (M5): `Registry::dispatch` echoes the tool signature + received keys on serde BadArgs ("missing field"/"invalid type" only); read_file translates directory paths to a list_files hint; grep falls back to literal search on invalid regex (header carries the reason, also on 0 matches). `run_command` runs `sh -c`/`cmd /C` from project root in its own process group so timeout/cancel kills the whole tree (`kill -9 -<pgid>` / `taskkill /T /F`); UTF-8 with CP949 lossy fallback for Korean Windows consoles; output middle-truncated past 8000 bytes ‚Äî the exec mechanics live in `tools/exec.rs`, shared with eval's `check` execution. M11: Done results append one English provenance line when the command contains a real unquoted pipe (`has_unquoted_pipe` ‚Äî quote/escape-aware, `||` is not a pipe) so models don't read a pipeline's exit 0 (last command's) as test success; timeout/cancel paths get no note
- notes (M16): config `repo_notes` (default **true**); tool `update_repo_notes` writes schema-checked `.loco/notes/_root.md` and dir notes; certified mut-gate before code `edit_file`/`write_file`; finish order VERIFY ‚Üí NOTES_STALE ‚Üí accept; `update_repo_notes` is `is_mutating` but **not** on the VERIFY re-arm whitelist (only `edit_file`|`write_file`); thrifty templates on reject bodies only; `.loco/notes` is **not** task-protected. Measurement / arms / markers: design spec **¬ß5** (`docs/superpowers/specs/2026-07-21-m16-repo-onboarding-design.md`); exp_metrics MARKS `notes_schema_reject`/`notes_mut_gate`/`notes_stale_finish`/`notes_updates` + column `notes_bytes_max`. GPU control/treatment not measured yet (experiment stub `docs/experiments/2026-07-21-m16-repo-onboarding/`)
- agent: confirmation gate lives inside the turn loop via the `Approver` trait (`TtyApprover`/`AutoApprover`/`NonInteractiveApprover`) ‚Äî fires only for `is_mutating()` tools with a successful `preview()`; denial feeds back to the model as a tool result. Turn parsing is salvage-tolerant (`protocol.rs` builds turns via `serde_json::Value`): fields outside `args` (action-level scalars, `args_2`, top-level non-reserved keys) are normalized into `args` (objects overwrite, scalars insert-if-absent; BTreeMap key order makes it deterministic), `ModelTurn.salvaged` triggers a one-line correction note merged into the tool result ‚Äî this drove gemma's M5 jump (285 salvages/36 runs). Repetition detection (`agent/repetition.rs`, M5): a `(tool|args, result-hash)` 8-turn window counted AFTER dispatch (denied results and summary-less `finish` included) ‚Äî 3rd identical hit injects one correction, 5th returns `AgentOutcome::RepetitionStop` (exit 2, the tool result is still pushed first); a separate same-error streak (first sentence, 3 consecutive) injects one strategy correction (edit_file/write_file ‚Üí "rewrite with write_file", else generic). Both corrections latch once per run. M9 scaffolding: edit_file's identical-search/replace rejection now appends a prescription sentence (current code in `search`, changed code in `replace`); a dedicated 2-consecutive S/R streak correction (`SR_CORRECTION`, own latch, `agent/repetition.rs`) handles that error key exclusively ‚Äî the generic 3-streak correction is excluded for it, while the window-based `REPEAT_CORRECTION` still stacks at the 3rd identical call (intended escalation). finish discipline: a missing-summary finish 2-consecutive streak injects `FINISH_ARGS_CORRECTION` once per run (no-action turns preserve the streak; dispatched or gate-denied actions reset it); `agent/finish_nudge.rs` is an event-driven state machine ‚Äî "mutation" = successful edit_file/write_file dispatch, arms on a post-mutation `exit code: 0` run_command, counts armed-state read/grep/list_files and re-verification run_command turns in a K=4 window requiring ‚â•1 repeated call (`RepetitionTracker::seen_key`, queried before record), injects `FINISH_NUDGE` once per run; RepetitionStop takes priority on the same turn. A summary-carrying `finish` after unverified mutations is rejected once per run (`VERIFY_NUDGE`; a `run_command` Ok clears the flag regardless of exit code except an empty test run ‚Äî see M12 below).
