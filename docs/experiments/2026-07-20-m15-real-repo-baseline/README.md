## 5. 다른 사람에게 설명할 때

### 30초 버전

1. loco는 로컬 소형 모델 코딩 CLI다.  
2. 합성 벤치는 거의 만점대라 **변별력이 사라졌다**.  
3. 그래서 **진짜 오픈소스 이슈 17개**를 만들어 돌렸다.  
4. **이슈 글만** 주고 레포 설명은 안 줬다.  
5. 결과: **0/51 · 전 과제 전패** — 테스트가 깨진 게 아니라 **cold start에선 못 푼다**.  
6. 다음: 레포를 먼저 요약하게 돕는 **온보딩 하네스**를 설계한다.

### 자주 나오는 오해

| 오해 | 정정 |
|---|---|
| “0점이라 실험 실패” | 효과 실험이 아님. **바닥 측정**이 목적. 실격은 “대조용 베이스라인 실패” 라벨 |
| “테스트가 전부 잘못됐다” | `--verify` 변별·해결 통과. solution 있으면 초록 |
| “`tasks/` 94%인데 왜?” | **다른 트랙**. 경로·규모·프롬프트 축이 다름 (M13 파일럿과 같은 교훈) |
| “M9–M14 하네스가 소용없다” | 실레포에선 **편집 기회 전**에 죽는 런이 많음. 장치 기회 분모가 다름 |
| “pack이 핵심 성과” | 부록. 본전은 좌표·실패 분해·트랙 |

### 숫자 인용 규칙

- 통과율: **이 배치 스탬프**만. M13 파일럿·`tasks/`와 나란히 “몇 %” 비교 금지.
