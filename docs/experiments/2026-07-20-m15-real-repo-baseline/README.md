### 얻지 못한 것 (기대하면 안 되는 것)

| 아님 | 이유 |
|---|---|
| “몇 %p 올리는 개입 승자” | 암 비교 없음 |
| M13/M14/`tasks/` 와 통과율 나란히 비교 | 채점·과제·트랙이 다름 (스펙 §8) |

### 중간 집계에서 본 행동 힌트 (휴리스틱, 공식 pool 전)

작성 직전 스냅샷 예 (로그+트랜스크립트, ~46런):

| 지표 | 대략 |
|---|---|
| nav (오라클 파일 read) | ~70%대 |
| fix (오라클 edit) | ~20%대 |
| zero-mut (성공 편집 0) | ~70%대 |
| pack 발동 런 | 소수 (rg 위주) |

**해석:** 파일 “탐색”만의 문제가 아니라, **레포 지도를 고정하지 못한 채 수선에 못 들어감**.

### pack만 얻은 거 아니냐?

| pack | 그 이상 |
|---|---|
| 대형 런에서 **발동 확인** (계측·로드 설계 헛일 아님) | **바닥 0 + 실패 분해 + 실격 + 인프라 + M16 방향** 이 본전 |
| 통과율 기여는 **관측 안 됨** | |

## 5. 다른 사람에게 설명할 때

### 30초 버전

1. loco는 로컬 소형 모델 코딩 CLI다.  
2. 합성 벤치는 거의 만점대라 **변별력이 사라졌다**.  
3. 그래서 **진짜 오픈소스 이슈 17개**를 만들어 돌렸다.  
4. **이슈 글만** 주고 레포 설명은 안 줬다.  
5. 결과: **거의 전부 실패 (0점대)** — 테스트가 깨진 게 아니라 **이 조건에선 못 푼다**.  
6. 다음: 레포를 먼저 요약하게 돕는 **온보딩 하네스**를 설계한다.

### 자주 나오는 오해

| 오해 | 정정 |
|---|---|
| “0점이라 실험 실패” | 효과 실험이 아님. **바닥 측정**이 목적. 실격은 “대조용 베이스라인 실패” 라벨 |
| “테스트가 전부 잘못됐다” | `--verify` 변별·해결 통과. solution 있으면 초록 |
| “`tasks/` 94%인데 왜?” | **다른 트랙**. 경로·규모·프롬프트 축이 다름 (M13 파일럿과 같은 교훈) |
| “M9–M14 하네스가 소용없다” | 실레포에선 **편집 기회 전**에 죽는 런이 많음. 장치 기회 분모가 다름 |
| “pack이 핵심 성과” | 부록. 본전은 좌표·실패 분해·트랙 |

### 숫자 인용 규칙

- 통과율: **이 배치 스탬프**만. M13 파일럿·`tasks/`와 나란히 “몇 %” 비교 금지.  
- 비율 환산 금지 구간: 파일럿 n, 중간 집계 휴리스틱.  
- 최종 인용: B5 완료 후 pool + `baselines.md` M15 절을 우선.
